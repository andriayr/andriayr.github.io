One of the most important and useful tools I became aquainted with was version control software.What does it do, you may ask... well simply put, it keeps track of the changes that you make to a file. Below you will find my summary of my two favorite benefits of version control (thus far) 1. Version control keeps track of your files, and logs all of the changes. This is awesome because if you accidently mess something up, you can always go back to your previous code before you broke your code. This is the perfect safeguard for any and all coders. 2. It lets you collaborate with others. Version control lets you work on the same file at the same time, without screwing up your code. Theres a feature called merge, that takes commits by two or more people and it merges the commits, and makes a new document that reflects the outcome of your collaboration.

How does git help you keep track of changes?

So far in my learning process there are a few key commands that have helped me understand how git helps me track files. To begin, the following commands should be embedded in your memory: 1. Make a new file: touch newfile.txt 2. Add it: git add newfile.txt 3.Make an initial commit: git commit -m "Initial commit" 4. See what you have done: git log 5. Check to see what has changed: git diff Its a good idea to commit early, and commit often. Commit is the equivalent of "save". If your commits are small, it will become much easier to go back and review what you have done, and allow you to undo the damage you may have caused. So to answer the question, how does git help you track changes, is by letting you commit early and often, and by letting you track all of the changes you made to you program.

Why use GitHub to store your code?

A few great reasons to use Github to store your code include: 1. Version Control- Everything you have on Github is stored on Git. And Version control is what allows you to make mistakes in your code without compromising the final product. 2.Keeps your code in a remote location: If you happen to be traveling or working on multiple computers, GitHub lets you access the codes you have been working on from anywhere. 3. Allows for Collaboration: Once you have pushed your code onto Github, you can invite others to work on your code, ask for help to debug or even ask for help if you get stuck.
