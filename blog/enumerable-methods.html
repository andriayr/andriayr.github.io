Understanding Maps

The map method is a very power enumberable in Ruby. Enumerables gives you many useful ways of doing something to every element of a collection of objects (arrays or hashes).This is necessary when you begin building programs, even those that are not very advanced.

Enumerable is a large bunch of methods, and there many to choose from. However, you will only use a dozen of them regularly. When you want to learn about more, there is a full list is available in the docs here.

#Map is a method that returns a new array filled with whatever gets returned by the block each time it runs.If no block is given, an enumerator is returned instead.

Below are some code samples that can illustrate the beauty of #map.

arr= [1,2,3,4,5]

arr.map { |a| 2*a } #=> [2, 4, 6, 8, 10]

arr #=> [1, 2, 3, 4, 5]

arr.map! { |a| a**2 } #=> [1, 4, 9, 16, 25]

arr #=> [1, 4, 9, 16, 25]

Above you will notice that there are that I have map and map! As you can see using map prints the original value of the array when it is called. However, when we used map! we changed the original value of the array. This difference is called destructive vs. non-destructive. While the destructive operations will modify the array they were called on, the non-destructive methods usually return a new array with the selected elements, but leave the original array unchanged.
